# Skills

Each immediate subdirectory here is a Claude Code skill (a `SKILL.md`, plus
any supporting files, per Claude Code's skill format).

Running `dev-env` symlinks each one into `~/.claude/skills/<name>`, so
skills can be authored/edited either here or in place under
`~/.claude/skills/` and stay in sync. Skills installed in
`~/.claude/skills/` from elsewhere (e.g. a marketplace) are left alone —
only names that exist here are managed.
